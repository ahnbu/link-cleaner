# Plan: 다크 테마 적용 및 UI 최적화

불필요한 중복 버튼을 제거하고, 전체적인 UI를 다크 테마로 전환하여 세련된 사용자 경험을 제공합니다.

## User Review Required

> [!IMPORTANT]
> - 우측 상단 헤더에 '새로 시작하기' 버튼이 이미 존재하므로, 결과 화면 툴바의 '새로 시작하기' 버튼을 제거합니다.
> - `index.html`에 설정된 `dark` 클래스와 `bg-slate-950` 배경색에 맞춰 모든 컴포넌트의 색상 스키마를 다크 테마로 변경합니다.

## Proposed Steps

### 1. UI 요소 제거
- `App.tsx`의 결과 화면 툴바(`Actions Toolbar`)에서 '새로 시작하기' 버튼 코드 제거.
- 버튼 제거 후 남은 '클립보드 복사' 및 '다운로드' 버튼의 레이아웃이 어색하지 않도록 조정.

### 2. 다크 테마 적용 (App.tsx)
- 메인 제목(`h2`) 및 설명(`p`)의 색상을 다크 테마에 맞게 변경 (`text-slate-800` -> `text-slate-100` 등).
- 로딩 상태 오버레이의 배경색 및 텍스트 색상 조정.
- 결과 툴바의 배경색을 다크 톤으로 변경 (`bg-white/90` -> `bg-slate-900/90`).

### 3. 컴포넌트 다크 테마 최적화
- `FileUploader.tsx`: 드롭존 배경 및 텍스트 색상 조정.
- `MarkdownPreview.tsx`: 카드 배경색(`bg-white` -> `bg-slate-900`) 및 텍스트 색상 조정.
- `Header.tsx`: (이미 일부 반영되었으나) 전체적인 일관성 확인.

## Verification Plan

### Automated Tests
- `npm run build`: 빌드 시 타입 에러나 정적 분석 에러가 없는지 확인.

### Manual Verification
- 브라우저에서 배경과 텍스트가 다크 모드에서 충분한 대비를 가지는지 확인.
- 결과 화면에서 상단 툴바가 여전히 고정(sticky)되어 동작하는지 확인.
- 툴바에서 '새로 시작하기' 버튼이 제거되었는지 확인.
