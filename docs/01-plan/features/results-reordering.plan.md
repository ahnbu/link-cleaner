# [Plan] 결과 화면 컴포넌트 재배치 (Results View Reordering)

## 1. 개요 (Overview)
사용자가 변환 결과물을 즉시 확인하고 활용할 수 있도록 결과 화면의 요소 순서를 중요도 순(액션 버튼 -> 정리된 내용 -> 원본 내용)으로 재배치합니다.

## 2. 변경 사항 (Changes)
- **순서 변경**: 
    1. 액션 바 (복사, 다운로드 등)
    2. 정리된 내용 (Cleansed Content)
    3. 원본 내용 (Original Content)
- **위치 조정**: 액션 바가 상단으로 이동함에 따라 `sticky` 위치를 `bottom-8`에서 `top-24` 정도로 조정하거나 플로우 내에 자연스럽게 배치하여 접근성을 높입니다.

## 3. 작업 목록 (Tasks)
- [ ] `App.tsx` 결과 렌더링 영역의 컴포넌트 순서 변경
- [ ] 액션 바의 `sticky` 위치 및 스타일 최적화
- [ ] 레이아웃 간격(spacing) 조정

## 4. 예상 결과 (Expected Outcome)
- 변환 완료 후 사용자가 가장 먼저 필요로 하는 기능(복사/다운로드)과 결과물을 최상단에서 바로 확인 가능.
- 비교를 위한 원본 데이터는 하단으로 이동하여 화면의 복잡도를 낮춤.
