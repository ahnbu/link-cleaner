# [Plan] 자동 클렌징 프로세스 및 UX 개선 (Auto-Cleansing UX Improvement)

## 1. 개요 (Overview)
사용자의 작업 단계를 줄이고(자동 실행), 직관적인 결과 확인(세로 2단 구성) 및 간결한 피드백(복사 완료 알림 개선)을 제공하여 서비스의 사용성을 극대화합니다.

## 2. 요구 사항 (Requirements)
- **자동 클렌징**: 파일 업로드 직후 별도의 클릭 없이 자동으로 클렌징 서비스를 호출.
- **UI 구조 단순화**: 옵션 설정 단계를 생략(기본값 사용)하고, 한 화면에서 업로드와 결과를 모두 처리.
- **세로형 레이아웃**: 결과 화면에서 오리지널과 클렌징 텍스트를 세로로 배치하여 비교를 용이하게 함.
- **다운로드 형식 수정**: 마크다운 다운로드 시 확장자를 `.md`로 고정.
- **복사 피드백 개선**: `alert()` 대신 버튼의 텍스트나 아이콘이 변경되는 방식의 인라인 피드백 제공.

## 3. 해결 방안 (Proposed Solutions)
- **App.tsx**: 
    - `handleFileUpload` 내에서 `handleCleansing`을 직접 호출하도록 수정.
    - `CleansingOptions` 컴포넌트 노출 제거.
    - `isProcessed` 상태에 따른 레이아웃을 1열 세로 스택 구조로 변경.
    - `handleDownload`의 파일명 확장자 로직 점검.
    - `handleCopy`의 상태 관리(예: `isCopied`) 추가 및 UI 연동.
- **FileUploader.tsx**: 업로드 후 로딩 상태를 직관적으로 표시.

## 4. 작업 목록 (Tasks)
- [ ] 파일 업로드 시 자동 클렌징 트리거 구현
- [ ] 옵션 설정 UI 제거 및 기본값 고정
- [ ] 결과 화면 레이아웃 변경 (세로 2단 스택)
- [ ] 다운로드 파일 확장자 `.md`로 수정
- [ ] 복사 버튼 인라인 피드백 UI 구현

## 5. 예상 결과 (Expected Outcome)
- 사용자는 파일 선택 한 번으로 결과물을 바로 얻을 수 있음 (2단계 -> 1단계).
- 불필요한 팝업 없이 매끄러운 사용자 경험 제공.
- 비교가 쉬운 직관적인 뷰포트 구성.
